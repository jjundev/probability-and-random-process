# 문제 PNG 정제(refine) 방법 기록

> 대상: `기말고사 대비/6월 *일/*.png` (문제 이미지 111장)
> 구현: `python/refine_pngs.py` (자동 파이프라인) + 일부 수동 처리
> 목적: 교재에서 잘라온 문제 PNG의 여백·레이아웃 문제를 정리해, 가로 PDF(`문제_6월N일.pdf`)에서 한 문제씩 깔끔하게 보이도록 함.

---

## 1. 배경: 왜 정제가 필요했나

문제 PNG는 두 교재(IPSRP=Pishro-Nik, PSP)에서 잘라낸 것이라 다음 문제가 있었다.

- **IPSRP**: 가로로 매우 넓고(≈1650px) 세로로 짧음 → 그대로 PDF에 넣으면 폭에 맞춰 축소되어 위쪽에 얇게 깔림.
- **PSP**: 한 단(컬럼) 너비(≈730px)·세로로 김. **2단 조판**이라 한 문제가 좌/우 컬럼에 걸쳐 잘리는 경우가 많음.

PDF로 만들었을 때 사용자가 본 구체적 증상 3가지:

| 케이스 | 증상 | 예시 |
|---|---|---|
| **C1 균일 측여백** | 문제가 한쪽(보통 왼쪽)으로 큰 여백을 두고 몰림 | `4.2.1` 좌측 233px 빈공간(우측 컬럼 문제라서) |
| **C2 2단 레이아웃** | (a)(b)는 왼쪽, (c)(d)는 오른쪽 → **아래쪽 왼쪽이 빔** | `4.4.6`, `1.3.4` |
| **C3 dirty crop** | 그래프·MATLAB 코드·해설·난이도 범례 등 **다른 요소가 같이** 잘림 | `10.1.2`, `5.1.2` |

---

## 2. 자동 파이프라인 (`refine_pngs.py`)

이미지 1장당 처리 순서:

```
원본(_png_원본백업) → ① content-bbox autocrop → ② 컬럼 reflow → 재-autocrop → 흰 여백(pad) 추가 → 저장
                                                                 → ③ dirty 후보 판정(보고만)
```

### 파라미터
| 이름 | 값 | 의미 |
|---|---|---|
| `THR` | 240 | gray < 240 이면 잉크(ink), 그 외 배경(흰색) |
| `K` | 3 | 한 행/열에서 잉크 픽셀이 이 수 이상이어야 "내용"으로 인정(노이즈 가드) |
| `PAD` | 14 | 크롭 후 사방에 두는 흰 여백(px) |
| `GAPBAND` | 22 | 가로 밴드를 나누는 빈 행 연속 길이(px) |
| reflow 임계 | `max(120, 0.22·W)` | 밴드 좌단이 본문 좌단보다 이만큼 이상 오른쪽이면 "딴 컬럼"으로 보고 끌어옴 |
| `GAP_FLAG` | 70 | 이미지 내부 세로 빈공간 갭이 이 값 이상이면 dirty 후보로 표시 |

### ① content-bbox autocrop (`bbox` / `crop_tight`)
- 그레이스케일로 변환 → `ink = gray < THR`.
- **행별** 잉크합 ≥ K 인 행, **열별** 잉크합 ≥ K 인 열만 내용으로 보고, 그 최소/최대로 bounding box 계산.
- 그 box로 crop → **바깥 여백만 제거**(내부 들여쓰기·수식 정렬은 그대로 보존).
- C1(균일 측여백) 완전 해결. 예: `4.2.1` 좌측 233px 제거.

```python
ink = gray < 240
rows = ink.sum(axis=1) >= 3      # 행별 잉크 ≥ K
cols = ink.sum(axis=0) >= 3      # 열별 잉크 ≥ K
l,t,r,b = xs.min, ys.min, xs.max, ys.max  # → crop
```

### ② 컬럼 reflow (`reflow`)
2단 레이아웃(C2)에서 **오른쪽 컬럼으로 밀려난 가로 밴드를 좌측 본문 마진으로 끌어온다.** 핵심은 **세로 위치를 보존**(밴드를 통째로 평행이동)해서 수식·줄간격이 안 깨지게 하는 것.

1. 가로 밴드 분리: 빈 행이 `GAPBAND`(22px) 이상 연속되면 밴드 경계.
2. 본문 좌단 `gmin` = 전체에서 가장 왼쪽 잉크 x.
3. 각 밴드의 좌단이 `gmin + max(120, 0.22·W)` 보다 오른쪽이면 → 그 밴드 전체를 `(밴드좌단 − gmin)`만큼 **왼쪽으로 평행이동**.
4. 이동 후 오른쪽에 생긴 빈공간은 재-autocrop으로 제거.

- **큰 이동량(임계 초과)만** 옮기므로, 살짝 들여쓴 수식(정상)은 건드리지 않음.
  - `1.3.4`: (c)(d)가 좌측 정렬로 이동 → 1단으로 정리 ✅
  - `4.4.6`: (a)(b) 이동, 수식 중괄호 정렬 보존 ✅
  - `4.2.1`: 이동량이 작아 그대로 → 수식 들여쓰기 보존 ✅(망가지지 않음)

### ③ dirty 후보 판정 (`internal_gap`)
- 첫 내용 행과 마지막 내용 행 사이의 **최대 연속 빈 행 길이**를 잼.
- `GAP_FLAG`(70px) 이상이면 "다요소/조각 혼입 의심"으로 `_재크롭_후보.md`에 기록(자동 수정은 안 함).
- 단, **검수 완료 allowlist**(`REVIEWED_CLEAN`)에 든 파일은 제외(오탐 방지, §4 참조).

---

## 3. 안전장치

- **원본 백업**: 최초 1회 각 PNG를 `_png_원본백업/6월 N일/`에 복사. 정제는 항상 **백업에서 읽어** 처리하므로, 다시 돌려도 결과가 같고(멱등) 언제든 되돌릴 수 있음. (교재 은행 `problems_by_lecture/`도 그대로라 2중 백업.)
- **멱등성**: 이미 타이트한 이미지는 bbox가 변화 없음, 이미 좌측정렬된 밴드는 reflow가 변화 없음 → 재실행해도 동일.
- **원자적 저장**: 임시 파일에 쓰고 교체.

## 4. dirty crop 수동 처리 (자동으로 못 고치는 8장)

`internal_gap ≥ 70`으로 뜬 8장을 **눈으로 확인**해 셋으로 분류·조치했다.

### (a) 진짜 dirty 2장 → 재크롭
교재 2단/페이지 경계 때문에 **진짜 문제(상단) + 이물(중간: 그래프·MATLAB·해설·난이도범례) + 진짜 문제 마지막 파트(하단)** 구조.
→ 행투영으로 블록 좌표를 구해, **진짜 블록만 골라 세로로 다시 쌓고(stack)** 이물은 버림. 결과를 백업에 덮어쓴 뒤 파이프라인 재처리.

```python
# 유지 블록(원본 y범위) — 나머지(그래프/코드/해설/범례)는 버림
10.1.2 : keep [(18,353), (1220,1306)]   # 본문(a~d) + CLT 마지막파트
5.1.2  : keep [(18,289), (1237,1262)]   # 본문(a~d) + (e)
```
각 블록을 `crop_tight`로 좌측정렬해 잘라낸 뒤 22px 간격으로 수직 결합. "Moderate/Difficult/Experts Only" 난이도 범례 줄도 제거.

### (b) 시험범위 밖 1장 → 교체
- `복습L02_PSP_2.5.1` = **MATLAB 시뮬레이션 문제**(시험 무관) → `2.4.2`(조건부확률, in-scope)로 교체. D8 `오늘_복습.md`도 갱신.

### (c) 오탐 5장 → 유지 + allowlist
완결된 정상 문제인데 문단/구분선 여백 때문에 갭만 컸던 것:
`L08_PSP_7.6.3`, `L09_IPSRP_7.3.2`, `7.3.4`, `7.3.5`, `L06_PSP_5.5.1`.
→ `refine_pngs.py`의 `REVIEWED_CLEAN` 집합에 등록해 재실행해도 다시 안 뜨게 함.

## 5. 설계 결정 / 폐기한 접근 (왜 이렇게 했나)

- **블록 좌측정렬 + 세로갭 정규화 후 재조립(restack) → 폐기.** 교재 행간 갭이 커서(최대 98~116px) 한 문제가 10블록으로 과분할되고, 정상 들여쓰기(예 `4.2.1` 블록 좌단 233/276/238)가 파괴됨이 실측으로 확인됨. 대신 **세로 위치를 보존하는 평행이동(reflow)** 채택.
- **dirty 판정 기준: 좌단편차/종횡비 → 폐기, 내부 세로갭 채택.** 좌단편차는 정상 수식 들여쓰기까지 잡아 111장 중 59장 오탐. 세로갭(≥70px) 기준으로 바꿔 8장으로 좁힘(그 중 5장은 추가 검수로 오탐 판명).
- **이물(그래프 등) 자동 삭제 안 함.** 여백이 아니라 "내용"이라 어떤 게 진짜 문제인지는 사람이 판단 → 후보 목록만 뽑고 수동 재크롭.

## 6. 한계

- **밴드 내부 2단**(같은 행에 좌·우 내용이 동시에 있는 경우)은 reflow가 못 고침(현재 사례엔 없음).
- dirty crop의 "진짜/이물" 구분은 자동화하지 않음(수동).
- reflow가 중앙 정렬 수식을 좌측정렬로 바꿀 수 있음(드묾, 가독성엔 무해).

## 7. 재실행 방법

```powershell
# 1) PNG 정제 (백업에서 읽어 재처리, 멱등)
python python/refine_pngs.py
# 2) 정제된 PNG로 문제 PDF 재생성
python python/build_daily_problem_pdfs.py
```

- 문제 세트(폴더의 PNG)가 바뀌면 위 2개를 다시 실행.
- 새 dirty 후보가 뜨면 `_재크롭_후보.md` 확인 → §4 방식으로 수동 처리.

## 8. 검증 방법

- 결과 PNG/ PDF 페이지를 직접 열어 눈으로 확인(특히 재크롭·reflow 대상).
- 확인 예: `4.2.1`(C1)·`1.3.4`/`4.4.6`(C2)·`10.1.2`/`5.1.2`(C3) 모두 1단으로 깔끔히 정리됨을 확인.
