## 문제
Example 2.21의 셀룰러폰 코딩 시스템을 오류를 줄이도록 수정한다. 구체적으로, 수신된 5비트 시퀀스에 0이 두 개 또는 세 개 있으면 삭제(deletion, 사건 D)가 일어났다고 한다. 그렇지 않고 0이 적어도 4개 수신되면 수신기는 0이 전송됐다고 판정하고, 1이 적어도 4개 수신되면 1이 전송됐다고 판정한다. i가 전송됐는데 수신기가 j ≠ i로 판정하면 오류(error, 사건 E)가 발생했다고 한다. 이 수정된 프로토콜에서 오류 확률 P[E]와 삭제 확률 P[D]는 얼마인가?

## 1. 문제 정리
모델(Example 2.21 / Problem 2.3.1): 5비트 코드워드의 각 비트는 서로 독립이며 **0일 확률 0.8, 1일 확률 0.2**. 0이 전송된 상황을 기준으로 하면, 비트가 0이면 정확 수신, 1이면 비트 오류(비트오류율 0.2).
Z = 수신된 5비트 중 0의 개수라 하면 Z ~ Binomial(5, 0.8).

판정 규칙(Z 기준):
- Z ∈ {4, 5} → "0 전송"으로 판정 (정답)
- Z ∈ {2, 3} → 삭제 D
- Z ∈ {0, 1} → "1 전송"으로 판정 (0을 보냈으므로 오류 E)

구할 것: P[E], P[D].

## 2. 무엇을 묻고 왜 이 도구인가
각 비트가 독립이고 동일 확률(0 일 확률 0.8)이므로 "0의 개수"는 n=5, 성공확률 p=0.8인 **이항분포**다. (Lecture03, p.11; 독립 시행의 곱은 Lecture02 곱셈법칙 p.11) 세 판정 영역은 Z의 값에 따른 배반 사건이므로, 각 영역 확률은 해당 이항 PMF 항들의 합이다.

## 3. 핵심 통찰
세 결정(정답/삭제/오류)은 오직 0의 개수 Z 하나로 결정되고 Z의 값 {0,1,2,3,4,5}를 배반적으로 분할한다 — 그러니 이항 PMF 여섯 항만 계산하면 모든 답이 나온다.

## 4. 풀이

이항 PMF: P[Z=k] = C(5,k)·(0.8)^k·(0.2)^(5−k),  k = 0,1,...,5,  otherwise = 0.

각 항:
P[Z=0] = (0.2)^5 = 0.00032
P[Z=1] = 5·(0.8)·(0.2)^4 = 5·0.8·0.0016 = 0.0064
P[Z=2] = 10·(0.8)^2·(0.2)^3 = 10·0.64·0.008 = 0.0512
P[Z=3] = 10·(0.8)^3·(0.2)^2 = 10·0.512·0.04 = 0.2048
P[Z=4] = 5·(0.8)^4·(0.2) = 5·0.4096·0.2 = 0.4096
P[Z=5] = (0.8)^5 = 0.32768

**오류 확률** (0을 보냈는데 1로 판정 = 0이 0개 또는 1개):
P[E] = P[Z ∈ {0,1}] = 0.00032 + 0.0064 = 0.00672

**삭제 확률** (0이 2개 또는 3개):
P[D] = P[Z ∈ {2,3}] = 0.0512 + 0.2048 = 0.256

## 5. 검산·직관
- 전체 PMF 합: 0.00032 + 0.0064 + 0.0512 + 0.2048 + 0.4096 + 0.32768 = 1 ✓.
- 세 영역 합: P["0"판정] + P[D] + P[E] = (0.4096+0.32768) + 0.256 + 0.00672 = 0.73728 + 0.256 + 0.00672 = 1 ✓.
- 직관: 비트오류율 0.2면 5비트 중 0이 대부분 나올 가능성이 높아 정답 판정 확률 0.737로 가장 큼. 4개 이상이 1로 뒤집혀야 오류가 나므로 P[E]가 0.0067로 매우 작은 게 타당 ✓. 삭제(애매한 2~3개)는 그 중간 0.256 ✓.
- 수정 목적("오류 감소")과 일치: 애매한 경우를 오류로 확정하지 않고 삭제로 빼내 P[E]를 낮춤.

## 6. 한 줄 요약
> 5비트 각각이 독립으로 0일 확률 0.8이니 0의 개수는 Binomial(5,0.8); 판정은 이 개수의 배반 구간들이라 P[E]=P[Z≤1]=0.0067, P[D]=P[Z∈{2,3}]=0.256으로 이항 항만 더하면 끝.
