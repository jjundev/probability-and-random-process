## 문제
각 코드워드에 5개의 비트(0 또는 1)가 있는 이진 부호를 고려하자. 코드워드의 예는 01010이다. 각 코드워드에서 한 비트는 다른 비트와 독립적으로 확률 0.8 로 0이다.
(a) 코드워드 00111 의 확률은?
(b) 코드워드가 정확히 세 개의 1을 포함할 확률은?

## 1. 문제 정리
- 주어진 것: 코드워드 = 5비트. 각 비트 B_i 는 독립이고
  - P(B_i = 0) = 0.8,  P(B_i = 1) = 1 − 0.8 = 0.2.
- 구할 것:
  - (a) P(00111) — 특정 비트열 하나의 확률.
  - (b) P(정확히 1이 3개) — 위치 무관, 1이 정확히 3개인 코드워드.

각 비트의 PMF (베르누이):

  P(B_i = 0) = 0.8
  P(B_i = 1) = 0.2
  P(B_i = b) = 0,   otherwise (b ∉ {0,1})

## 2. 무엇을 묻고 왜 이 도구인가
비트들이 서로 독립이므로, 특정 비트열의 확률은 각 비트 확률의 곱이다 — 독립의 정의 P(A∩B) = P(A)·P(B)의 직접 적용 (Lecture02, p.19). (b)는 "1이 3개"인 비트열이 여러 개(자리 선택) 있으므로, 같은 확률을 갖는 배타적 비트열의 개수 C(5,3)을 곱한다 — 이항계수 세기.

## 3. 핵심 통찰
독립이면 비트열 확률 = 비트 확률의 곱이고, 1의 개수만 같으면(위치만 다르면) 모든 비트열의 확률이 같다(0.2^k · 0.8^(5−k)). 그래서 "정확히 k개"의 확률은 (자리 경우의 수) × (한 비트열 확률) = C(5,k)·0.2^k·0.8^(5−k).

## 4. 풀이

(a) 비트열 00111: 0이 2개, 1이 3개. 독립이므로 곱 (Lecture02, p.19, 독립의 정의):

  P(00111) = P(0)·P(0)·P(1)·P(1)·P(1)
           = 0.8 × 0.8 × 0.2 × 0.2 × 0.2
           = 0.8^2 × 0.2^3
           = 0.64 × 0.008
           = 0.00512

(b) 정확히 1이 3개. "1이 3개"인 코드워드 각각의 확률은 위치와 무관하게 0.2^3 · 0.8^2 = 0.00512 로 동일((a)에서 확인). 5자리 중 1이 들어갈 3자리를 고르는 경우의 수 = C(5,3) = 10, 이들은 서로 배타적이므로 더한다:

  P(정확히 1이 3개) = C(5,3) × 0.2^3 × 0.8^2
                   = 10 × 0.008 × 0.64
                   = 10 × 0.00512
                   = 0.0512

(이 세기 단계의 일반식 C(n,k)p^k(1−p)^(n−k)는 이항분포 PMF — ⚠ 이항 PMF 자체는 Lecture02 정리노트에 명시 없음(독립까지만 다룸). 독립 곱은 Lecture02, p.19에서 인용했고, 자리 세기는 조합 일반지식으로 보강.)

## 5. 검산·직관
- (b) = 10 × (a): "1이 3개"인 비트열 중 하나가 정확히 00111이고, 그런 비트열이 10개라는 점이 (a)와 (b)의 관계로 드러남 — 일관성 ✓.
- 직관: 0이 1보다 4배 흔하므로(0.8 vs 0.2) 1이 3개나 되는 코드워드는 드물다(5%대) — 합리적.
- 전체 점검(보너스): Σ_(k=0..5) C(5,k)0.2^k 0.8^(5−k) = (0.2+0.8)^5 = 1 ✓ (이항정리).

## 6. 한 줄 요약
> 독립 비트는 비트열 확률 = 비트확률 곱(00111 → 0.8^2·0.2^3=0.00512)이고, "정확히 3개"는 자리 선택 C(5,3)=10을 곱해 0.0512다.
