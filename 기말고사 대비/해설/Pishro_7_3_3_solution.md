## 문제
어떤 통신 시스템에서 각 코드워드(codeword)는 1000비트로 구성된다. 잡음 때문에 각 비트는 확률 0.1로 오류가 발생할 수 있다. 비트 오류는 서로 독립적으로 일어난다고 가정한다. 이 시스템은 오류정정부호(error correcting code)를 쓰므로, 수신된 코드워드의 오류가 125개 이하이면 신뢰성 있게 복호되고, 그렇지 않으면 복호에 실패한다. CLT를 사용하여 복호 실패 확률을 구하라.

## 1. 문제 정리
- 주어진 것: 각 비트의 오류 여부 X_i ~ Bernoulli(p=0.1), i.i.d., n = 1000.
  - E[X_i] = p = 0.1, Var(X_i) = p(1−p) = 0.1 × 0.9 = 0.09.
- Y = 코드워드의 총 오류 비트 수 = X_1 + … + X_1000 ~ Binomial(1000, 0.1).
- 구할 것: 복호 실패 확률 = P(Y > 125) = P(Y ≥ 126).

## 2. 무엇을 묻고 왜 이 도구인가
n = 1000개 i.i.d. 베르누이의 합(이항)의 꼬리확률 문제. 이항 꼬리 Σ C(1000,k)(0.1)^k(0.9)^(1000−k) 를 k=126부터 1000까지 직접 더하는 것은 비현실적이다. 강의노트: 합의 분포는 컨볼루션이라 어려우니 표준화해 가우시안으로 근사 (Lecture09, slide 5, slide 9). 따라서 CLT가 정답 도구다.

## 3. 핵심 통찰
Y = ΣX_i 는 평균 np, 분산 np(1−p)을 가지며, (Y − np)/√(np(1−p)) 가 N(0,1)로 수렴한다 — 베르누이여도 합을 표준화하면 가우시안. (Lecture09, slide 9)

## 4. 풀이
합의 평균·분산 (i.i.d. 단순 합) — (Lecture09, slide 4):

  E[Y] = np = 1000 × 0.1 = 100
  Var(Y) = np(1−p) = 1000 × 0.09 = 90
  σ(Y) = √90 ≈ 9.4868

CLT 표준화 (Lecture09, slide 9):

  Z = (Y − 100)/9.4868  ≈  N(0,1)

복호 실패 = P(Y > 125):

  P(Y > 125) = P( Z > (125 − 100)/9.4868 )
             = P( Z > 25/9.4868 )
             = P( Z > 2.6352 )
             = 1 − Φ(2.6352)

표준정규 분포표에서 Φ(2.64) ≈ 0.99585 이므로

  P(Y > 125) ≈ 1 − 0.99585 = 0.00415  ≈  0.0042

(문제가 연속성 보정을 명시하지 않았으므로 위 값을 표준 답으로 둔다. 보정을 쓰면 P(Y ≥ 126) ≈ P(Z > (125.5 − 100)/9.4868) = P(Z > 2.688) ≈ 0.0036.)

## 5. 검산·직관
- 경계: "125 이하면 성공, 초과면 실패" → 실패는 Y > 125, 즉 Y ≥ 126. 표준화 임계값을 125로 잡은 것이 맞다(보정 미적용 기준).
- 직관: 평균 오류 100개인데 정정 한계는 125개로 약 2.64σ 위 → 실패는 0.5% 미만으로 드물다. 오류정정부호가 충분한 여유(margin)를 준다는 설계 의도와 일치.
- 단위: 오류 비트 수(정수)의 표준화 → 무차원. ✓

## 6. 한 줄 요약
> 이항(1000, 0.1) 오류 수의 꼬리는 평균 np=100·분산 np(1−p)=90으로 표준화해 1 − Φ((125−100)/√90) ≈ 0.0042 로 근사한다.
