## 문제
50 km 사이클 타임트라이얼에서 선수의 정확한 기록 T (분 단위)는 연속균등 (50, 60) 확률변수다. 그러나 초 단위로 기록되는 R 은 T 를 **다음 정수 초로 올림(round up)** 하여 얻는다. 즉 T 가 50 분 27.001 초이면 R = 3028 초이고, T 가 정확히 50 분 27 초이면 R = 3027 초다. R 의 PMF 는?

## 1. 문제 정리
- 주어진 것: T ~ Uniform(50, 60) 분. R = (T 를 초로 환산한 뒤 다음 정수 초로 올림).
- 구할 것: 이산확률변수 R 의 PMF P_R(r).

## 2. 무엇을 묻고 왜 이 도구인가
연속 RV 에서 이산 RV 를 만드는 derived distribution. 원칙은 "특정 출력값이 되는 입력 영역의 확률"을 모으는 것 — 이산 출력이므로 P_R(r) = P(R = r) 를 입력 사건으로 환산 (Lecture07, p.6 의 이산 derived 합산 사고). 올림(ceiling)은 "어떤 T 구간이 같은 r 로 가는가"를 따지는 문제다.

## 3. 핵심 통찰
올림 R = r 이 되는 것은 정확히 **T_초 ∈ (r−1, r]** 일 때다 (r−1 초 바로 위부터 r 초 정각까지는 모두 r 로 올림). T 가 600 초 길이의 균등분포라, 길이 1 초짜리 각 구간이 똑같이 확률 1/600 을 갖는 **이산균등** R 이 나온다.

## 4. 풀이

### 단위 환산
T ~ Uniform(50, 60) 분을 초로: 50 분 = 3000 초, 60 분 = 3600 초.
⇒ T_초 ~ Uniform(3000, 3600), 즉 f(t) = 1/600 for 3000 < t < 3600, otherwise = 0. (구간 길이 600.)

### 올림 사건 환산
정수 r 에 대해 R = ⌈T_초⌉ = r ⟺ r − 1 < T_초 ≤ r.
(예시 확인: T_초 = 3027.001 ∈ (3027, 3028] → r = 3028 ✓; T_초 = 3027 정각 ∈ (3026, 3027] → r = 3027 ✓.)

### r 의 범위와 확률
T_초 가 (3000, 3600) 에 있으므로 r − 1 ≥ 3000 과 r ≤ 3600 → r ∈ {3001, 3002, …, 3600} (총 600 개).
각 r 에 대해 해당 T_초 구간 길이는 1 초이고 (양 끝 점은 확률 0 이라 무시 가능), 균등밀도 1/600:
P_R(r) = P(r−1 < T_초 ≤ r) = 1 × (1/600) = 1/600.

따라서

P_R(r) = 1/600,  r = 3001, 3002, …, 3600
P_R(r) = 0,       otherwise

(R 은 {3001,…,3600} 위의 이산균등.)

## 5. 검산·직관
- 정규화: 값 개수 3600 − 3001 + 1 = 600 개 × (1/600) = 1 ✓.
- 경계 직관: T 의 최솟값 3000 초(정각)는 확률 0 이라 r = 3000 은 안 나오고, 바로 위 (3000, 3001] 이 가장 작은 올림값 3001 을 준다 → 하한이 3001 인 것이 타당. 상한은 60 분 = 3600 초 정각까지 올림되어 3600.
- 올림 vs 반올림 구분: 올림이라 R ≥ T_초 항상 성립 — 기록이 실제보다 결코 짧게 표기되지 않음(시간 측정의 보수적 처리)과 일치.

## 6. 한 줄 요약
> 연속균등을 정수로 올림하면 각 정수 r 은 길이-1 입력구간 (r−1, r] 에 대응해 동일 확률을 받아, R 은 이산균등 P_R(r) = 1/600 (r = 3001…3600) 이 된다.
